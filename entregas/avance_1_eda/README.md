# Avance 1. Analisis exploratorio de datos

## Proyecto

**AURORA: Analisis Unificado de Riesgo Operativo y Readiness con Aprendizaje Automatico**

AURORA es un caso de estudio academico que utiliza datos sinteticos para explorar senales genericas de riesgo y preparacion de liberaciones en flujos empresariales de software. En este avance se realiza el analisis exploratorio de datos para entender estructura, distribuciones, relaciones, faltantes, atipicos y cardinalidad antes de avanzar hacia ingenieria de caracteristicas y modelado.

## Entregable

- Nombre del entregable: `Avance1.4`
- Modalidad: individual
- Equipo: 4
- Archivo principal: `Avance1.4.ipynb`

## Archivos

- `Avance1.4.ipynb`: libreta principal con el analisis exploratorio de datos.
- `data/release_readiness_synthetic.csv`: dataset ficticio generado de forma sintetica.

## Privacidad

El dataset es completamente sintetico. No contiene datos reales de Oracle ni informacion confidencial. Las variables son genericas y fueron creadas para demostrar el proceso de EDA sin exponer sistemas, tecnologias, clientes, procedimientos, repositorios, URLs, credenciales ni metricas reales.

## Contenido cubierto

La libreta cubre:

- estructura, tipos de datos, estadisticos descriptivos, frecuencias y valores faltantes;
- analisis univariante con histogramas, boxplots y barras;
- analisis bi/multivariante con correlaciones, comparaciones contra variable objetivo y tendencia temporal sintetica;
- preprocesamiento de faltantes, atipicos y alta cardinalidad;
- conclusiones principales del EDA.

## Ejecucion

Ejecutar `Avance1.4.ipynb` de forma secuencial desde la primera celda. El archivo CSV debe permanecer dentro de la carpeta `data/`.
